# SansarPay First 10 Users

This is the exact 7-day manual launch checklist for the first 10 SansarPay users.

Use this file if you are the founder running the first beta by hand.

Goal:

- invite 10 real users
- help them complete the core flow
- learn where trust breaks
- decide whether to continue, fix, or pause feature work

## Target User Profile

Target users should look like this:

- Nepali worker in Qatar
- Nepali worker in UAE
- Nepali worker in Malaysia
- Nepali worker in Saudi
- Nepali worker in Saudi Arabia
- sends money home monthly
- family uses WhatsApp
- can reply on WhatsApp without much delay
- willing to test a simple app and give direct feedback

Do not start with random users.
Start with users who already send money home and already use WhatsApp with family.

## 7-Day Launch Plan

### Day 1: Invite 10 Users

Today’s goal:

- send WhatsApp invites to 10 target users
- confirm who is interested
- write each person into the manual tracking table

Checklist:

- Pick 10 users who match the target profile.
- Spread them across Qatar, UAE, Malaysia, and Saudi if possible.
- Send the first WhatsApp invite.
- Mark who replied.
- Mark who said yes.

End of day check:

- 10 invites sent
- at least 5 positive replies
- all 10 names entered in the tracking table

### Day 2: Help Signup

Today’s goal:

- help users complete signup
- clear OTP issues quickly

Checklist:

- Follow up with every user who said yes.
- Ask them to open the app while you are available on WhatsApp.
- Help them enter the correct phone number.
- Confirm they finish signup.
- If OTP fails, use the debug loop immediately.

End of day check:

- as many users as possible signed up
- OTP problems written down clearly
- no user left hanging without a reply

### Day 3: Get 5 Transfers Logged

Today’s goal:

- get at least 5 real transfers logged

Checklist:

- follow up with signed-up users
- ask them to log their latest transfer
- confirm amount, date, and method are entered
- note where users get confused

End of day check:

- 5 transfers logged
- main friction points written down

### Day 4: Get 3 Goals Created

Today’s goal:

- get at least 3 users to create a savings goal

Checklist:

- ask users with a logged transfer to set one goal
- help them choose a simple goal name
- help them enter a target amount and monthly amount
- check whether the commitment screen makes sense to them

End of day check:

- 3 goals created
- note whether users understand why the goal matters

### Day 5: Get 3 Family Links Opened

Today’s goal:

- get at least 3 family links shared and opened

Checklist:

- ask users to copy the family link from the app
- ask them to send it to wife, mother, father, or another family member
- ask the family member to open the link on WhatsApp
- confirm the link actually opened

End of day check:

- 3 family links shared
- 3 family links opened
- note whether family understood what the page meant

### Day 6: Call Every User

Today’s goal:

- speak to each user directly
- understand trust, confusion, and repeat intent

Checklist:

- call or voice-message every signed-up user
- ask the founder call questions below
- write answers in the feedback column
- mark repeated trust problems

End of day check:

- every reachable user contacted
- repeat problems grouped by theme

### Day 7: Summarize Learnings

Today’s goal:

- turn the first week into a clear decision

Checklist:

- count signups
- count transfers logged
- count goals created
- count family links opened
- group user feedback into patterns
- decide whether to continue, fix, or pause feature work

End of day check:

- one short written summary exists
- top 3 problems are clear
- next decision is clear

## Founder Call Questions

Ask these exactly.
Keep the call short and direct.

### Core Questions

- What confused you?
- Did you trust the app?
- Did your family understand the link?
- Would you use this next month?

### Useful Follow-Up Questions

- Which step felt slow?
- Which screen was hard to understand?
- Did anything feel risky or suspicious?
- Did WhatsApp sharing work easily?
- What would stop you from using it again?

## Daily Founder Routine

Use this every day:

1. Check the tracking table in the morning.
2. Message users who are stuck.
3. Help one step at a time on WhatsApp.
4. Update the table immediately after each reply.
5. Log bugs the same day.
6. Do not wait until later to remember details.

## Decision Rules

Use these rules after Day 7.

### Continue

Continue onboarding more users if:

- 10 users signed up or close to it
- 5 transfers were logged
- 3 goals were created
- 3 family links were opened
- users mostly trust the app
- the same bug is not blocking multiple users

### Fix

Stop new onboarding and fix issues first if:

- users sign up but cannot complete the next step
- users do not trust the app flow
- family links are confusing or fail often
- OTP problems repeat
- one trust bug appears in more than one user

Fix the smallest broken area only.
Preserve working code.

### Pause Feature Development

Pause feature work if:

- core flows are still unstable
- users are confused by basic signup or transfer logging
- trust is low
- family members do not understand the link
- the founder is spending most time supporting bugs instead of learning from usage

If this happens:

- stop adding new features
- fix core issues only
- re-run the first-user launch loop after the fixes

## What Success Looks Like

At the end of 7 days, you should know:

- whether workers can sign up without heavy help
- whether they will log real transfers
- whether they understand savings goals
- whether family link sharing works in real life
- whether users trust SansarPay enough to use it next month

If you do not know those five things clearly yet, the launch week is not finished.
